How It Works

From GitHub API to
your inbox in minutes

Async parallel processing across hundreds of repositories with zero infrastructure required beyond GitHub Actions.

01 🔑

Authenticate

Configure your GitHub PAT with repo, security_events, and read:org scopes.

02 ⚡

Collect

Async parallel calls to GitHub GraphQL & REST APIs aggregate Dependabot, Code Scanning, Secret Scanning, and Supply Chain data.

03 📊

Analyze

Risk scoring, trend analysis, and compliance checks surface what matters. Vulnerabilities are ranked by severity, age, and impact.

04 📋

Report

Multi-tab Excel workbooks are generated automatically — Daily, Weekly, or on-demand Org Inventory — and emailed to your team.

05 🔄

Automate

GitHub Actions workflows run on schedule with no additional infrastructure. Zero-maintenance once configured.

Quick Start

Up and running
in three steps

Clone, configure, and generate your first report. GitHub Actions handles everything after that.

1

Install & Run

# Clone and install
$ git clone https://github.com/alien-c0de/
    github-security-reporter.git
$ pip install -r requirements.txt

# Run reports
$ python main.py daily
$ python main.py weekly
$ python main.py orgdata
2

Configure .env

# GitHub (Required)
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
GITHUB_ORG=your-org-name
GITHUB_ENTERPRISE_URL=your-ent-url
# Branding
COMPANY_NAME=Your Company
SUPPORT_EMAIL=team@company.com
# Email (Optional)
EMAIL_TO=security@company.com
3

Automate with GitHub Actions — add secrets and enable workflows

# Workflows in .github/workflows/
# daily-summary.yml   → runs every day at 07:00 UTC
# weekly-report.yml   → runs every Monday at 08:00 UTC

# Required repo secrets:
ORG_SECURITY_TOKEN   # PAT with security_events + read:org
EMAIL_USERNAME        # SMTP username
EMAIL_PASSWORD        # SMTP app password
